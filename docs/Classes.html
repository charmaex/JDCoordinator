Classes

The following classes are available globally.

  • AppCoordinator

    Use the AppCoordinator only in a UINavigationController based application.

    The coordinator structure in your app can be seen as a tree. In this example the AppCoordinator is the root of every other coordinator. In order to prevent a coordinator from beeing deinitialized it has to have a strong reference somewhere. The AppCoordinator should be referenced by the AppDelegate. Every other Coordinator should be referenced by the AppCoordinator or one of its Childs.

    Use this class only once in your app. It should be the Coordinator where every navigation is started from. The AppCoordinator has to use the UINavigationController which the AppDelegate declares as keyWindow.

    See more

  • Coordinator

    The Coordinator is the simplest class in a UINavigationController based application.

    The coordinator structure in your app can be seen as a tree. In this example the Coordinator is the end of any branch. A Coordinator can only manage UIViewControllers and should not instantiate or strongly reference any other coordinator.

    See more

    Declaration

    Swift

    open class Coordinator: Coordinating, Navigating, MutableChild, ControllerDelegate, StartTestable, Identifiable
  • ParentCoordinator

    Use the ParentCoordinator only in a UINavigationController based application.

    The coordinator structure in your app can be seen as a tree. In this example the ParentCoordinator is a branch between the root and a leave. A ParentCoordinator can manage other Coordinators as childs and also manage UIViewControllers (usage of a Child is preferred). It is totally safe to add another ParentCoordinator as a child.
